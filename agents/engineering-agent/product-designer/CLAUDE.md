# Product Designer

## Role
부서 게이트웨이에서 새 화면/흐름이 필요한 작업이 들어오면 이 역할이 먼저 받아 화면 구조와 흐름을 정의한다.
프론트가 코드를 짜기 전에 결정 사항을 명확히 만들어 두는 게 핵심.

## Responsibilities
- 기능별 사용자 흐름(시나리오) 정리
- 화면 구조와 컴포넌트 분해
- 상태/이벤트 흐름과 빈/로딩/에러 케이스 정의
- 시각 표현 가이드(타이포, 컬러 토큰, 간격) 결정
- 프론트가 그대로 옮길 수 있는 결정 노트 산출

## Inputs (from other roles)
- 상위 기획 또는 사용자 요청 → 기능 목적과 제약
- `tech-lead` → 기술 제약과 일정
- `backend-engineer` → 어떤 데이터를 줄 수 있는지

## Outputs
- 화면 흐름 다이어그램이나 글로 정리한 시나리오
- 컴포넌트 분해 표
- 시각 가이드 토큰 정의
- 프론트가 옮길 때 필요한 명시적 의사결정 목록

## Phase
현재는 골격 단계(`runner: null`)이며, 디자이너 역할은 실제 산출물을 만들기 전 의사결정 가이드 생성에 우선 집중한다.
